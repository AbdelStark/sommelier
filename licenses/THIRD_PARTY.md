dataset, and runtime packages that the Sommelier reference pipeline consumes.

before publishing any derived artifact.

## Hebrew v3 Forward Translation Provider

- **Provider and API identity**: OpenAI Responses,
  `gpt-5.5-2026-04-23`, with explicit Flex service.
- **Terms boundary**: access and generated output are governed by the OpenAI
  terms and data controls applicable to the account at request time. No OpenAI
  model weights are downloaded, distributed, or assigned an open-source license
  by this repository. Re-verify the current provider terms before publishing a
  derived dataset.
- **Use in this project**: one-time English-to-Hebrew query translation through
  the strict `instruction_chat` contract. The CPU producer pins Python 3.13.3,
  `openai==2.45.0`, and `datasets==5.0.0`; requests use at most 512 output
  tokens, explicit Flex service, reasoning effort `none`, `store=false`, no
  background mode, disabled truncation, a 900-second per-request timeout, and
  zero SDK retries. Keeping SDK retries at zero makes every repeated call a
  visible source-id/attempt event owned by the row pipeline, which permits three
  audited attempts with eight provider workers and 32-row durable checkpoints.
- **Data boundary**: protected literals are placeholder-masked before the API
  request. The source query and the bounded selected-tool name/description and
  parameter name/type/description projection are still processed by OpenAI.
  Tools and gold answers remain byte-identical in the paired rows. `store=false`
  is not a Zero Data Retention claim.
- **Evidence boundary**: the dated model name is an API identity, not a public
  weight digest or a guarantee of byte-identical regeneration. The raw v2
  provider journal remains in durable producer artifacts; the published
  translation summary includes its digest and a content-free v2 aggregate of
  returned identity, counts, usage, and a calculated public-list-price estimate.
  That estimate is not an invoice or observed billing. Accepted rows remain a
  machine-translated survivor corpus, not a native translation release.
- **Publication policy**: disclose the external one-time teacher and publish
  the exact dated snapshot, service tier, client/request identity, journal
  digest/aggregate, audited drop counts, semantic-review boundary, and
  machine-translation limitation with every derived Hebrew dataset release.

## Hebrew Semantic-Review Back-Translation Model

- **Artifact**: `Helsinki-NLP/opus-mt-tc-big-he-en`, pinned to
  `134c5a850dcaa763eec85bd1f4eb25112fecedbb`.
- **License (per the model card)**: CC-BY-4.0.
- **Attribution**: Helsinki-NLP, OPUS-MT project, `opus-mt-tc-big-he-en`.
- **Source card**:
  <https://huggingface.co/Helsinki-NLP/opus-mt-tc-big-he-en>
- **Requested scholarly citations from the model card**: Jörg Tiedemann and
  Santhosh Thottingal, *OPUS-MT -- Building open translation services for the
  World* (EAMT 2020), and Jörg Tiedemann, *The Tatoeba Translation Challenge*
  (WMT 2020).
- **Use in this project**: preregistered greedy Hebrew-to-English
  back-translation for the locked 200-row semantic-review sample through the
  Transformers Marian path. It differs from the forward translator and is not
  selected or tuned after inspecting review judgments. Source tokenization is
  non-truncating with a hard 512-token limit and batches are capped at eight.
- **Upstream benchmark context**: the model card self-reports BLEU 44.1 on
  FLORES-101 devtest and 53.8 on `tatoeba-test-v2021-08-07`. Sommelier has not
  independently reproduced those scores, and they do not measure semantic
  review accuracy on this project.
- **Publication policy**: publish both the untouched machine template and the
  finalized semantic-review artifact. Together they record this exact revision,
  decoding contract, license, locked sample/input digests, non-native reviewer
  boundary, and whole-publication pass/fail decision. The model is not
  redistributed by this repository.
